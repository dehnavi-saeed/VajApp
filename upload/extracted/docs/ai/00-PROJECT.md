# VAJ Project Constitution

Version: 3.0

---

# Project Identity

Project Name

VAJ

Project Type

Reading & Knowledge Management Platform

Target Users

Individuals who want to build a personal knowledge base through reading books.

Primary Goal

Help users transform books into structured, searchable and long-lasting knowledge.

---

# Vision

VAJ is designed to become a lifelong companion for readers.

The platform helps users read intentionally, organize knowledge, remember what they learn and continuously improve their understanding.

Reading is not the final goal.

Learning is.

Knowledge is.

Long-term personal growth is.

---

# Mission

Enable people to:

- Build personal libraries
- Read physical and digital books
- Track reading progress
- Highlight important passages
- Create structured notes
- Connect ideas together
- Organize knowledge
- Review previous learning
- Achieve reading goals
- Measure learning progress

---

# Core Principles

Knowledge First

Every feature must increase the value of the user's knowledge.

Reading Before Productivity

Reading experience is more important than feature count.

Long-term Thinking

Knowledge created today should still be useful years later.

Simple by Default

Complexity must never be introduced without a strong reason.

Production Ready

Every implementation must be maintainable, testable and scalable.

AI Friendly

The project structure must be understandable by both humans and AI coding assistants.

---

# What VAJ Is

VAJ is:

- A reading platform
- A personal knowledge system
- A learning assistant
- A book management platform
- A note-taking system
- A highlight management system
- A reading analytics platform
- A long-term knowledge repository

---

# What VAJ Is NOT

VAJ is NOT:

- ERP
- CRM
- Accounting software
- Warehouse management
- Inventory management
- Payroll system
- HR software
- Hospital software
- School management system
- Generic CMS
- E-commerce platform

Never generate features outside the reading domain unless explicitly requested.

---

# Core Values

Every design decision should support at least one of the following:

- Better reading
- Better learning
- Better knowledge organization
- Better recall
- Better usability
- Better maintainability

If a feature does not improve one of these areas, reconsider its necessity.

---

# Primary Actors

- Guest
- User
- Administrator

Future actors may be introduced only when justified by business requirements.

# Success Metrics

The success of VAJ is measured by:

- Books completed
- Reading consistency
- Notes created
- Highlights reviewed
- Knowledge retained
- User engagement
- Learning continuity

Not by the number of implemented features.

---

# Long-Term Philosophy

Knowledge is permanent.

User-created knowledge must never be accidentally lost.

Every design decision should prioritize:

1. Data integrity
2. User trust
3. Simplicity
4. Maintainability
5. Scalability

---

# AI Constitution

Every AI working on VAJ must understand that:

- VAJ is a Reading Platform.
- Books are the central domain object.
- Knowledge is more important than CRUD.
- Reading workflows are more important than technical elegance.
- Maintainability is preferred over clever implementations.
- New features must align with the project vision.
