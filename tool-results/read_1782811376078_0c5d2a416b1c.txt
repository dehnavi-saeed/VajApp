   195‚Üí# Domain Rules
   196‚Üí
   197‚ÜíThe Domain is the most important layer.
   198‚Üí
   199‚ÜíBusiness rules belong only here.
   200‚Üí
   201‚ÜíThe Domain has zero framework dependencies.
   202‚Üí
   203‚ÜíNever import:
   204‚Üí
   205‚ÜíSpring
   206‚Üí
   207‚ÜíHibernate
   208‚Üí
   209‚ÜíJackson
   210‚Üí
   211‚ÜíMapStruct
   212‚Üí
   213‚ÜíJPA
   214‚Üí
   215‚ÜíLombok
   216‚Üí
   217‚Üíinside Domain.
   218‚Üí
   219‚Üí---
   220‚Üí
   221‚Üí# Aggregate Rules
   222‚Üí
   223‚ÜíEvery Aggregate has one Aggregate Root.
   224‚Üí
   225‚ÜíAggregate Root protects invariants.
   226‚Üí
   227‚ÜíRepositories load only Aggregate Roots.
   228‚Üí
   229‚ÜíNever expose child Entities directly.
   230‚Üí
   231‚ÜíPrefer small Aggregates.
   232‚Üí
   233‚ÜíReference external Aggregates only by Identifier.
   234‚Üí
   235‚Üí---
   236‚Üí
   237‚Üí# Entity Rules
   238‚Üí
   239‚ÜíEntities have identity.
   240‚Üí
   241‚ÜíEntities own behavior.
   242‚Üí
   243‚ÜíEntities never become DTOs.
   244‚Üí
   245‚ÜíEntities never become database models.
   246‚Üí
   247‚ÜíNever expose Entities through REST.
   248‚Üí
   249‚Üí---
   250‚Üí
   251‚Üí# Value Object Rules
   252‚Üí
   253‚ÜíImmutable.
   254‚Üí
   255‚ÜíNo identity.
   256‚Üí
   257‚ÜíEquality by value.
   258‚Üí
   259‚ÜíNo setters.
   260‚Üí
   261‚ÜíExamples
   262‚Üí
   263‚ÜíBookTitle
   264‚Üí
   265‚ÜíEmailAddress
   266‚Üí
   267‚ÜíISBN
   268‚Üí
   269‚ÜíReadingDuration
   270‚Üí
   271‚ÜíProgressPercentage
   272‚Üí
   273‚ÜíPrefer Value Objects over primitives.
   274‚Üí
   275‚Üí---
   276‚Üí
   277‚Üí# Repository Rules
   278‚Üí
   279‚ÜíRepository interfaces belong to Domain.
   280‚Üí
   281‚ÜíJpaRepository implementations belong to Infrastructure.
   282‚Üí
   283‚ÜíRepository responsibilities:
   284‚Üí
   285‚ÜíLoad Aggregate
   286‚Üí
   287‚ÜíSave Aggregate
   288‚Üí
   289‚ÜíNothing else.
   290‚Üí
   291‚ÜíNever implement business rules inside Repositories.
   292‚Üí
   293‚Üí---
   294‚Üí
   295‚Üí# Domain Service Rules
   296‚Üí
   297‚ÜíDomain Services exist only when behavior does not naturally belong to one Aggregate.
   298‚Üí
   299‚ÜíAvoid Service classes that only move logic out of Aggregates.
   300‚Üí
   301‚ÜíPrefer rich Domain Models.
